# Topic Discovery v4.1 更新计划

## 🎯 更新目标

基于 RED Phase 测试结果，v4.1 需要实现以下 4 个功能：

### 1. 增加"用户痛点"字段 ✅

**位置**：Step 4 - 生成选题建议

**修改内容**：
- 在选题字段表中增加"用户痛点"行
- 在输出模板中增加"用户痛点"字段
- 提供结构化的痛点描述格式

**格式要求**：
```markdown
**用户痛点**：
- 核心问题：[具体问题描述]
- 困扰原因：[为什么这个问题让用户困扰]
- 解决方案：[这个选题如何解决这个问题]
```

### 2. 增加"互动引导策略"字段 ✅

**位置**：Step 4 - 生成选题建议

**修改内容**：
- 在选题字段表中增加"互动引导策略"行
- 在输出模板中增加"互动引导策略"字段
- 提供结构化的互动策略格式

**格式要求**：
```markdown
**互动引导策略**：
1. **内容设计**：[如何在内容中埋下引导点]
2. **引导话术**：[文末或评论区的引导语]
3. **预期互动**：[读者可能的反应]
4. **回复策略**：[如何回复读者的询问]
```

### 3. 完善商品建议输出格式 ✅

**位置**：Step 4.5 - 商品关联（策略A：起号期）

**修改内容**：
- 确保输出完整的 9 个飞书表格字段
- 为每个选题生成 3-5 个商品建议
- 商品建议必须从选题内容中提取需求

**输出格式**：
```markdown
### 📦 商品建议（可直接录入飞书商品库）

#### 选题X：[选题标题]

**商品需求分析**：
- 核心需求：[从选题中提取的商品需求]
- 痛点：[用户在这个话题下的痛点]
- 优先推荐：[商品分类]

| 商品名称 | 商品分类 | 价格 | 库存 | 商品链接 | 爆款潜力 | 适用平台 | 创建时间 | 备注 |
|---------|---------|------|------|---------|---------|---------|---------|------|
| ... | ... | ... | ... | 待补充 | ... | ... | 2026-02-07 | ... |
```

### 4. 输出到飞书多维表格 ✅

**位置**：Step 5 - 生成报告并保存

**修改内容**：
- 使用 Feishu MCP 将选题输出到飞书多维表格
- 飞书表格路径：慢养四季运营数据库 > 选题清单
- 不再保存到本地 `/选题库/` 目录

**Feishu MCP 调用**：
```python
# 使用 mcp__feishu-mcp__feishu_add_content_record
for topic in topics:
    feishu_add_content_record(
        title=topic['选题标题'],
        platform=topic['推荐平台'],
        content_type="选题",
        date=today
    )
```
