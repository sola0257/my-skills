## 二、执行流程

### Step 1: 使用 AskUserQuestion 工具询问

**必须使用工具**: 不能跳过,不能用文本询问替代

**询问格式**:
```
问题: 您的[平台名称]账号当前粉丝数是多少?
选项:
- 0粉 (刚开始)
- 1-100粉
- 101-500粉 (起号期)
- 501-2000粉 (成长期)
- 2000+粉 (成熟期)
- 其他 (请输入具体数字)
```

**⚠️ 重要变更**:
- 不再询问"有没有变化"，而是询问"当前粉丝数"
- 每次都获取具体数字，才能判断变化趋势
- 用户选择区间后，AI应该记录区间中位数（如"1-100粉"记录为50）
- 用户选择"其他"后，必须输入具体数字

### Step 2: 读取历史粉丝数记录

**文件位置**: `/Users/dj/Desktop/全域自媒体运营/内容发布/粉丝数记录_{平台}.json`

**平台映射**:
- 小红书 → `粉丝数记录_小红书.json`
- 公众号 → `粉丝数记录_公众号.json`
- 视频号 → `粉丝数记录_视频号.json`

```json
    }

  "current_stage": "起号期"
}
```

### Step 3: 判断粉丝数变化

**情况A: 没有变化**
- 计算距离上次涨粉的天数
- 如果 ≥ 5天 → 触发5天提醒机制

**情况B: 有变化**
- 计算涨粉数量
- 更新记录文件
- 生成个性化鼓励语

### Step 4: 生成个性化鼓励语

**重要**: 必须使用 AI 动态生成,不是硬编码的固定文案

**生成规则**:
1. **根据具体情况生成**:
   - 涨粉数量 (涨了多少)
   - 坚持天数 (从第一条记录到现在)
   - 最近内容 (最近发布的内容主题)
   - 账号阶段 (起号期/成长期/成熟期)

2. **鼓励语风格**:
   - 真诚、具体、有温度
   - 避免空洞的"加油"、"继续努力"
   - 结合具体数据和成果
   - 提供下一步建议

3. **示例** (仅供参考,每次都应该不一样):
   ```
   太棒了!从0到150粉,你坚持了30天,平均每天涨5粉。
   上周的"多肉养护教程"表现不错,涨了10粉。
   建议:继续深耕养护类内容,这是你的优势方向。
   ```

### Step 5: 5天未涨粉提醒机制

**触发条件**: 距离上次涨粉 ≥ 5天

**提醒内容**:
```
⚠️ 注意:已经5天没有涨粉了

可能的原因:
1. 内容选题不够吸引人
2. 发布时间不够优化
3. 标题/封面不够吸睛
4. 内容质量需要提升

建议:
- 分析最近5天的内容数据
- 对比爆款内容的差异
- 调整选题方向或内容形式
```

**询问用户**:
```
是否需要我帮你分析最近的内容数据,找出问题所在?
选项:
- 是,帮我分析
- 不用,我自己调整
```

### Step 6: 更新粉丝数记录

**操作**:
1. 读取现有记录文件
2. 追加新记录
3. 更新 `last_update` 和 `current_stage`
4. 保存文件

**使用脚本**: `/Users/dj/Desktop/小静的skills/_shared_scripts/follower_tracker.py`

## 五、常见问题

**Q: 如果用户选择"没有变化",还需要生成鼓励语吗?**
A: 需要。但鼓励语的重点是"坚持",而不是"涨粉"。

**Q: 如果是第一次使用,没有历史记录怎么办?**
A: 创建新的记录文件,初始化第一条记录。

**Q: 5天提醒机制会不会太频繁?**
A: 不会。这是一个温和的提醒,不是强制要求。用户可以选择忽略。
