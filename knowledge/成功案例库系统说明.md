# 成功案例库系统说明

> **创建日期**：2026-01-29
> **目的**：建立跨平台共享但有优先级的配图生成知识库

---

## 📚 系统架构

### 核心理念

**"固定确认部分 + 只改需要改善的部分"**

通过记录经过多次迭代验证的成功 prompt，避免重复试错，提高配图生成效率。

### 文件结构

```
小静的skills/
├── knowledge/ (共享知识库)
│   ├── image-generation-successful-cases.md (主案例库)
│   └── 成功案例库系统说明.md (本文档)
│
├── wechat-content-generator/
│   └── knowledge/
│       └── wechat-image-prompt-guide.md (v1.3) ✅
│
├── xiaohongshu-content-generator/
│   └── knowledge/
│       └── image-prompt-guide.md (v4.2) ✅
│
└── video-script-generator/
    └── knowledge/
        └── video-image-prompt-guide.md (v2.0) ✅ 新增详细风格模板
```

**说明**：
- 成功案例库放在共享的 `knowledge/` 目录下，所有 Skill 都可以访问
- 各平台的配图指南通过相对路径引用：`../../knowledge/image-generation-successful-cases.md`
- 这样避免了重复维护，保持了单一数据源

## 🎯 使用规则

### 1. 优先级原则

```
生成配图时的查找顺序：
1. 查看本平台的成功案例（优先）
2. 如果没有，查看其他平台案例（需调整比例）
3. 如果都没有，使用基础模板
```

### 2. 案例标注规范

每个成功案例必须包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| **来源平台** | 案例最初验证的平台 | 微信公众号（订阅号长文） |
| **适用平台** | 可应用的平台列表 | 微信公众号（优先）、小红书（需调整） |
| **验证状态** | 验证情况 | ✅ 已验证 / 🔄 待验证 / ⚠️ 需要优化 |
| **迭代次数** | 经过多少次调整 | 6次 |
| **固定部分** | 不可修改的核心要素 | 颜色控制、植物健康描述等 |
| **可调整部分** | 可根据内容修改的部分 | 植物名称、场景位置等 |

### 3. 跨平台应用规则

**通用部分**（所有平台适用）：
- ✅ 颜色控制（muted, desaturated, dusty coral）
- ✅ 真实感约束（natural window light, film-like quality）
- ✅ 健康描述（healthy, vibrant but not artificial）
- ✅ 风格约束（NOT product photography, NOT studio lighting）

**需调整部分**（根据平台）：
- 📐 比例描述（2.35:1 / 16:9 / 3:4 / 9:16）
- 📏 尺寸规格（900×383 / 1080×1440 / 1080×1920）
- 🎨 构图方式（横版 vs 竖版）

## 📝 添加新案例的流程

### 1. 迭代验证

经过多次迭代（建议 ≥3次），确认 prompt 稳定可靠。

### 2. 记录案例

在 `image-generation-successful-cases.md` 中添加：

```markdown
## 案例 N：[标题]

### 📋 基本信息
- **来源平台**：[平台名称]
- **适用平台**：[平台列表]
- **验证状态**：✅ 已验证（日期）
- **迭代次数**：N次

### ✅ 最终确认的 Prompt
[完整 prompt]

### 🔒 固定部分（不要修改）
[核心要素]

### 🔄 可调整部分（根据具体内容修改）
[可变要素]

### 📊 迭代过程总结
[版本对比表]

### 🎓 关键经验
[经验总结]

### ⚠️ 常见错误
[错误示例和正确做法]
```

### 3. 更新引用

在相关平台的配图指南中添加引用链接。

## 🔄 维护机制

### 定期更新

- **每月回顾**：检查现有案例是否仍然有效
- **版本标注**：记录案例的最后验证日期
- **状态更新**：更新验证状态（✅ / 🔄 / ⚠️）

### 案例淘汰

如果某个案例：
- 连续3次使用效果不佳
- 平台规则发生变化导致不适用
- 有更好的替代案例

则标记为 ⚠️ 需要优化 或移除。

## 💡 最佳实践

### 1. 迭代时的策略

❌ **错误做法**：每次大幅修改整个 prompt
```
问题：导致之前调好的部分变差
```

✅ **正确做法**：识别确认好的部分，只修改需要改进的部分
```
策略：
1. 明确哪些部分已经满意（固定）
2. 明确哪些部分需要改进（调整）
3. 只修改需要改进的部分
4. 验证后更新固定部分
```

### 2. 记录的粒度

**固定部分**：
- 具体到关键词级别
- 说明为什么这些词重要
- 记录避免使用的词

**可调整部分**：
- 提供替换选项
- 说明调整范围
- 给出填空模板

### 3. 经验提炼

不仅记录"怎么做"，更要记录"为什么"：
- ✅ 真实感 ≠ 不健康
- ✅ 避免完美 ≠ 要有缺陷
- ✅ 现代感 ≠ 冷淡
