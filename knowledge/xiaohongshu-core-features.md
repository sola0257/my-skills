## 📌 使用说明

本文档定义小红书信任型内容的核心功能和规范。

**核心理念**: 从种草型升级为信任型,通过客观、真实、专业的内容建立长期信任。

---

## 🎯 信任型内容策略

### 核心变化对比

| 维度 | 种草型(旧) | 信任型(新) |
|------|-----------|-----------|
| **内容目标** | 吸引点击,提升曝光 | 建立信任,促进转化 |
| **标题策略** | 情绪化痛点标题 | 客观+信任导向标题 |
| **内容结构** | 产品优点展示 | 真实体验+问题解决 |
| **语气调性** | 营销感较强 | 客观、中性、专业 |
| **引流方式** | 直接引导添加 | 价值先行+自然引导 |

## 一、信任型双标题系统

> ⚠️ **重要**: 小红书的"双标题"指的是**视觉标题**(封面大字)和**正文标题**(点开笔记后的标题),两者必须不同!

**详细标题公式**: 参考 `../../knowledge/title-formulas-by-platform.md`

### 1. 视觉标题(封面大字)

**作用**: 在信息流里抓住用户眼球,激发点击欲

**字数建议**: 10-15字(封面显示效果最佳)

**[强制] 封面图文字必须使用视觉标题**:
- 封面图上叠加的文字 = 视觉标题
- 不能使用正文标题作为封面文字
- 视觉标题更短、更有冲击力、更情绪化

**信任型公式**: 客观描述 + 价值点 + 时间/经验

**示例**:
- ✅ "用了6个月的绿植养护心得,客观说说优缺点"
- ✅ "3年花园设计经验:新手的5个常见误区"
- ✅ "客观测评:这款网红植物到底值不值得买"
- ✅ "踩过坑才明白:绿植养护的3个关键错误"

**爆款型公式**(详见标题公式库):
- 公式一: 人群+痛点+解决方案
- 公式二: 数字+结果+反常识
- 公式三: 价值+互动+钩子

**标题选择决策树**:
- 起号期(0-500粉): 可以适度使用爆款型
- 成长期(500-2000粉): 信任型+爆款型混合
- 成熟期(2000+粉): 以信任型为主

**避免的旧式标题**:
- ❌ "绝了!这个方法让我家绿植爆盆"
- ❌ "姐妹们冲!绿植养护的秘密被我发现了"
- ❌ "不允许你还不知道的神奇养护法"

---

### 2. 正文标题(笔记标题)

**作用**: 包含搜索关键词,让平台推荐给精准用户

**[强制] 字数限制**: ≤20个字(小红书平台限制)

**公式**: 核心关键词 + ｜ + 具体内容描述

**SEO优化要求**:
- 必须包含核心搜索关键词
- 使用｜分隔主标题和副标题
- 控制在20字以内

**示例对比**:
| 视觉标题(封面) | 正文标题(笔记标题,≤20字) |
|----------------|-------------------|
| 养了2年多肉才明白:新手最容易犯的3个错误 | 多肉养护避坑｜新手3大错误 (13字) |
| 踩过坑才知道:绿萝这样养才能爆盆 | 绿萝养护技巧｜浇水光照配土全攻略 (16字) |
| 客观测评:这款网红植物到底值不值得买 | 琴叶榕养护经验｜3个月真实体验分享 (16字) |

---

### 3. 标签(笔记底部)

**[必需] 标签规则**: 核心关键词(5个) + 长尾词(5个)

**示例**:
```
#绿植养护 #阳台花园 #居家绿植 #植物养护技巧 #室内植物
#园艺新手 #花园设计 #绿植避坑 #植物养护经验 #生活美学
```

**检查点**: 生成标签时必须包含大标签(覆盖流量)和长尾词(精准匹配)

## 二、信任型正文框架

**[强制] 字数限制**: ≤1000个字(小红书平台限制)

**详细内容结构**: 参考 `../../knowledge/content-structures-short-form.md`

**内容方向**: 参考 `../../knowledge/content-directions-universal.md`
- 核心策略: 使用混搭玩法(颜值+求生、懒人+颜值、求生+懒人)
- 不要单独使用某一类内容方向

### 5步信任型内容结构

```
1. 开头(建立共鸣+权威背景)
"最近很多朋友问我关于[主题]的问题,
我在[相关领域]做了[时间],服务了[数量]客户,
今天客观分享一下我的经验和看法。"

2. 背景说明(增加可信度)
"其实我刚开始也踩了不少坑,
[具体经历],后来通过[学习/实践]才找到正确方法。"

3. 核心内容(价值输出)
"总结了几个关键点,都是我踩过坑得出的经验:
第1个:[问题描述] + [解决方案] + [注意事项]
第2个:[问题描述] + [解决方案] + [注意事项]
第3个:[问题描述] + [解决方案] + [注意事项]"

4. 客观总结(平衡评价)
"整体来说,这个方法[总体评价],
但也要注意[可能的不足],
大家可以根据自己的情况选择。"

5. 建议指引(引导行动)
"如果大家有具体问题,
可以在评论区留言,我会逐一回复。
需要更详细资料的朋友,评论说'要资料'~"
```

## 三、语气调性升级

### 推荐用词

- "客观来说……"、"个人体验是……"
- "根据我的经验……"、"需要注意的是……"
- "比较推荐的是……"、"可能比较适合……"
- "我发现……"、"建议大家……"

### 避免用词

- ❌ "必买!"、"绝了!"、"太爱了!"
- ❌ "姐妹们冲!"、"一定要买!"
- ❌ "我不允许你还不知道!"
- ❌ 过度感叹号和情绪化表达

### 私域引导优化

**价值先行引导**:
```
"我花了3个月整理的《[主题]完整攻略》包含:
✅ [具体内容1]
✅ [具体内容2]
✅ [具体内容3]
评论区说'要攻略' → 自动发送领取方式"
```

**自然过渡引导**:
```
"评论区留言的朋友,我都会回复详细建议,
也可以点击我主页的小程序,有更多资料供参考~"
```

## 五、信任型配图策略

### 配图基本要求

**数量**: 12-15张(封面1张 + 正文11-14张)
**尺寸**: 1080x1440像素(3:4竖版,小红书推荐)
**命名**: 根据图片类型区分,详见 `knowledge/image-prompt-guide.md`
**类型**: 对比图、步骤图、细节图

**⚠️ 强制执行**:
- 配图数量不得少于12张
- 配图尺寸必须为3:4竖版

**检查点**:
- [ ] 配图数量达到12-15张
- [ ] 所有配图尺寸为1080x1440
- [ ] 文件命名符合规范
- [ ] 包含对比图、步骤图、细节图

### 图片生成 Prompt 规则

**强制要求**:
- 尺寸: `3:4 vertical format` 或 `1080x1440 pixels`
- 文字: `NO TEXT, NO WORDS, NO LETTERS`(默认无文字)
- 如需文字: 必须中文,清晰锐利(`CLEAR CHINESE TEXT, SHARP and LEGIBLE`)

**详细指南**: 参考 `knowledge/image-prompt-guide.md`(包含完整的 prompt 模板、示例、风格选择规则、色彩要求)
