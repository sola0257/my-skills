> **目的**：记录经过实际验证的成功 prompt，避免重复试错

---

## 📖 使用规则

### 优先级原则
1. **优先参考本平台案例**：生成配图时，优先参考标注为目标平台的案例
2. **跨平台参考**：如果本平台没有合适案例，可以参考其他平台的案例，但需要调整比例和尺寸
3. **案例标注**：每个案例都会标注来源平台、适用场景、验证状态

### 案例标注说明
- **来源平台**：案例最初在哪个平台验证成功
- **适用平台**：案例可以应用到哪些平台（需要调整比例）
- **验证状态**：✅ 已验证 | 🔄 待验证 | ⚠️ 需要优化

## 🔧 使用建议

### 1. 生成前准备
- 确定目标平台（微信/小红书/视频号）
- 确定图片类型（封面/正文配图）
- 确定是否需要文字叠加

### 2. 选择案例
- 优先选择同平台、同类型的案例
- 如无完全匹配，选择最接近的案例
- 注意比例和尺寸的调整

### 3. 调整 Prompt
- 保持案例中的"固定部分"不变
- 只修改"可调整部分"
- 参考案例中的"常见错误"避免踩坑

### 4. 验证清单
每个案例都提供了验证清单，生成后逐项检查

## 📝 贡献新案例

### 案例命名规范
```
case-[编号]-[平台]-[类型].md

示例：
- case-001-wechat-cover.md
- case-002-xhs-cover.md
- case-003-xhs-scene.md
```

### 案例文档结构
每个案例文档应包含：
1. 基本信息（平台、类型、验证状态）
2. 最终确认的 Prompt
3. 固定部分（不要修改）
4. 可调整部分（根据内容修改）
5. 迭代过程总结
6. 关键经验
7. 常见错误
8. 使用建议
