# 统一内容生成工作流 v1.0

**创建日期**：2026-01-30
**适用范围**：所有内容生成类 Skills（wechat、xiaohongshu、video）

---

## 🎯 核心原则

1. **交互优先**：必须先进行苏格拉底式提问，收集完整信息
2. **去重检查**：必须检查已发布内容，避免选题重复
3. **阶段适配**：根据账号阶段决定商品植入策略
4. **流程统一**：所有内容生成技能遵循相同的执行流程

---

## 🔄 标准执行流程（8步法）

### Step 0: 苏格拉底式提问 [强制]

**使用工具**：`AskUserQuestion`

**必须询问的信息**：

| 问题 | 选项 | 说明 |
|------|------|------|
| 内容平台 | 小红书/微信订阅号/微信服务号/视频号/抖音/快手 | 确定平台类型 |
| 内容格式 | 长文/图文/短视频/中视频 | 确定内容形式 |
| 内容目的 | 涨粉/引流/推销商品/纯干货分享 | 确定内容策略 |
| 选题方向 | 用户指定/AI推荐 | 确定选题来源 |
| 商品植入 | 是/否/极少 | 确定是否需要商品 |

**特殊询问**：
- 如果是"微信订阅号图文" → 额外询问格式（诗意金句/实用干货/故事叙事/问答互动）
- 如果是"视频" → 额外询问时长（15秒/30秒/1分钟/3分钟）

---

### Step 1: 检查已发布内容 [强制]

**目的**：避免选题和角度重复

**操作步骤**：

1. **读取已发布内容目录**
   ```
   小红书：/Users/dj/Desktop/全域自媒体运营/内容发布/发布记录/2026/小红书/
   订阅号：/Users/dj/Desktop/全域自媒体运营/内容发布/发布记录/2026/订阅号/
   服务号：/Users/dj/Desktop/全域自媒体运营/内容发布/发布记录/2026/服务号/
   视频号：/Users/dj/Desktop/全域自媒体运营/内容发布/发布记录/2026/视频号/
   抖音：/Users/dj/Desktop/全域自媒体运营/内容发布/发布记录/2026/抖音/
   快手：/Users/dj/Desktop/全域自媒体运营/内容发布/发布记录/2026/快手/
   ```

2. **分析最近30天的内容**
   - 列出所有标题和主题
   - 统计话题频率（如：蝴蝶兰出现3次，仙客来出现2次）
   - 识别重复的角度（如：浇水、黄叶、烂根等）

3. **输出去重分析报告**
   ```
   已发布内容分析（最近30天）：
   - 蝴蝶兰养护：已发布3篇（浇水、黄叶、烂根）
   - 仙客来养护：已发布2篇（温度、浇水）
   - 年宵花选购：已发布1篇

   ⚠️ 建议避开：蝴蝶兰、仙客来、年宵花的常规养护话题
   ✅ 建议方向：其他植物品种、春季换盆、病虫害防治等
   ```

---

### Step 2: 确定选题 [强制]

**根据 Step 0 和 Step 1 的信息**：

**情况A：用户指定选题**
- 直接使用用户提供的选题
- 但仍需检查是否与已发布内容重复
- 如果重复，提醒用户并建议调整角度

**情况B：AI推荐选题**
1. 查询营销日历（使用 `/marketing-calendar`）
2. 结合去重分析，排除已发布的话题
3. 推荐3个选题供用户选择
4. 每个选题说明：时间节点、差异化角度、预期效果

**选题原则**：
- ✅ 与已发布内容有明显差异
- ✅ 符合当前时间节点（营销日历）
- ✅ 符合账号阶段（起号期避免过度商业化）
- ✅ 有搜索流量和实用价值

---

### Step 3: 知识搜索 [必需]

**操作步骤**：

1. **网络搜索**（使用 `WebSearch`）
   - 搜索最新的权威知识
   - 关键词：选题 + 2026 + 专业术语

2. **本地知识库搜索**（使用 `Grep`）
   - 路径：`/Users/dj/Desktop/全域自媒体运营/知识库/`
   - 查找相关的知识文档

3. **整合知识**
   - 网络知识 + 本地知识
   - 提取核心要点
   - 确保内容的权威性和准确性

---

### Step 4: 商品匹配 [条件执行]

**执行条件判断**：

| 账号阶段 | 粉丝数 | 是否执行商品匹配 | 植入策略 |
|---------|--------|----------------|---------|
| 起号期 | 0-500 | ✅ 执行（了解商品） | 仅作案例提及，禁止价格，禁止直接推荐 |
| 成长期 | 500-2000 | ✅ 执行 | 软植入，评论区引导，可提价格（客观对比） |
| 成熟期 | 2000+ | ✅ 执行 | 可挂链接，可说价格，可直接推荐 |

**执行条件**：
- 用户在 Step 0 中明确要求商品植入
- 或内容目的是"推销商品"

**起号期（0-500粉）详细规则**：
- ✅ 执行商品匹配（了解有哪些相关商品）
- ✅ 仅作案例提及（如："我自己用的是XX"）
- ❌ 禁止价格
- ❌ 禁止直接推荐
- ⚠️ 商品不主导内容方向，内容价值优先
- ⚠️ 内容比例：100%价值输出

**成长期（500-2000粉）详细规则**：
- ✅ 执行商品匹配
- ✅ 软植入（1-2段自然提及）
- ✅ 评论区引导
- ✅ 可以提价格（客观对比）
- ⚠️ 内容比例：80%价值 + 20%引导

**成熟期（2000+粉）详细规则**：
- ✅ 执行商品匹配
- ✅ 可挂链接
- ✅ 可说价格
- ✅ 可直接推荐
- ⚠️ 内容比例：70%价值 + 30%转化

**操作步骤**（如果执行）：
1. 调用 `/product-catalog` 匹配相关商品
2. 选择1-3个商品
3. 根据账号阶段生成对应的植入文案

**重要**：
- 商品匹配是为了了解有哪些相关商品，不是为了主导内容方向
- 内容价值永远优先，商品次要
- 起号期的"仅作案例提及"是指在讲解知识点时，自然提到自己使用的商品，而不是推荐

---

### Step 5: 生成内容 [必需]

**根据平台类型生成**：

**小红书**：
- 双标题（视觉标题 + 搜索标题）
- 正文（信任型内容，客观真实）
- 12-15张配图（3:4竖版）

**微信订阅号**：
- 标题 + 摘要 + 正文
- 长文：1封面（2.35:1）+ 3正文（16:9）
- 图文：1封面（3:4）+ 5正文（3:4）

**视频脚本**：
- 分镜脚本 + 口播文案
- 4平台封面（小红书/视频号/抖音/快手）
- 分镜配图

**内容质量要求**：
- 符合账号阶段的内容策略
- 符合平台调性
- 通过违禁词检查

---

### Step 6: 生成配图 [必需]

**操作步骤**：

1. **读取配图指南**
   - 小红书：`knowledge/image-prompt-guide.md`
   - 微信：`knowledge/wechat-image-prompt-guide.md`
   - 视频：`knowledge/video-image-prompt-guide.md`

2. **生成配图**
   - 调用对应的配图生成脚本
   - 确保尺寸、数量、风格符合平台要求

3. **文件命名**
   - 小红书：`序号_建议文字说明.png`
   - 微信：`cover.png`, `01.png`, `02.png`, `03.png`
   - 视频：`cover_xiaohongshu.png`, `shot_01.png`, etc.

---

### Step 7: 保存文件 [必需]

**文件结构**：

```
/Users/dj/Desktop/全域自媒体运营/内容发布/发布记录/2026/[平台]/
└── YYYY-MM-DD_选题名称/
    ├── YYYY-MM-DD_选题名称.md
    ├── cover.png
    ├── 01.png, 02.png, ...
    └── data.json
```

**data.json 内容**：
```json
{
  "date": "2026-01-30",
  "platform": "订阅号",
  "topic": "选题名称",
  "format": "长文",
  "purpose": "涨粉",
  "product_included": false,
  "status": "待发布"
}
```

### Step 8: 推送/发布 [必需]

**根据平台类型**：

| 平台 | 操作 | 脚本 |
|------|------|------|
| 微信订阅号/服务号 | 推送至草稿箱 | `wechat_publish.py` |
| 小红书 | 保存文件，手动发布 | - |
| 视频号/抖音/快手 | 保存文件，手动发布 | - |

## ✅ 检查清单

每次执行内容生成任务时，必须完成以下检查：

- [ ] 已完成苏格拉底式提问（Step 0）
- [ ] 已检查已发布内容，确认无重复（Step 1）
- [ ] 已确定选题，与已发布内容有明显差异（Step 2）
- [ ] 已执行知识搜索（Step 3）
- [ ] 已根据账号阶段决定是否商品匹配（Step 4）
- [ ] 已生成内容，符合平台要求（Step 5）
- [ ] 已生成配图，符合平台规格（Step 6）
- [ ] 已保存文件到正确目录（Step 7）
- [ ] 已推送/发布（Step 8）

---

## 🚫 禁止的行为

1. ❌ 跳过苏格拉底式提问，直接开始生成内容
2. ❌ 不检查已发布内容，导致选题重复
3. ❌ 在起号阶段过早进行商品匹配，限制内容范围
4. ❌ 不同平台使用不同的执行流程
5. ❌ 跳过任何必需步骤
