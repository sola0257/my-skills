## 🎯 核心原则

### 原则1：用户真实问题是基础，不是限制

**❌ 错误理解**：
- 看到用户问题清单，就只能从这8个问题里选
- 必须用清单里的原话做标题
- 不能偏离清单内容

**✅ 正确理解**：
- 用户问题清单是"焦虑地图"，不是"选题清单"
- 从焦虑点出发，可以延伸出多个角度
- 可以用更有趣的方式包装同一个问题

**示例**：

用户真实问题：**"为什么买回来就黄叶、掉叶、烂根？"**

可以延伸的选题：
1. 直接解决型：《为什么你家绿植总是黄叶？80%都是光照问题》
2. 工具型：《3个判断光照够不够的小方法，不用买测光仪》
3. 预防型：《买绿植前必问自己的3个问题，避免养死》
4. 对比型：《同样是绿萝，为什么别人家的绿油油，你家的黄兮兮？》
5. 场景型：《客厅角落/办公桌/卧室，分别适合什么植物？》
6. 反常识型：《你以为是缺水，其实是缺光——绿植黄叶的真相》

**关键**：都在解决"黄叶"问题，但角度不同、包装不同。

## 🔄 选题生成流程（标准化）

### Step 1: 读取三层信息

```
1. 读取最新用户问题监测（260130文件夹）
2. 查询营销日历（当前日期）
3. 检查已发布内容（最近30天）
```

### Step 2: 识别焦虑点 + 时间节点

**问自己**：
- 用户现在最焦虑什么？（从问题清单提取）
- 现在是什么时间节点？（从营销日历提取）
- 这两者有什么结合点？

**示例**：

```
用户焦虑：浇水频率不知道怎么判断
时间节点：1月31日，冬春交替
结合点：冬春交替期，浇水频率要调整，但用户不知道怎么判断土壤干湿
```

### Step 3: 生成3个候选选题（不同角度）

**必须包含3种不同类型**：

1. **直接解决型**：直接回答用户问题
   - 例：《冬春交替期，浇水频率怎么调？3个判断方法》

2. **工具/方法型**：提供具体工具或方法
   - 例：《不用买测湿仪，3个土办法判断该不该浇水》

3. **场景/对比型**：从场景或对比角度切入
   - 例：《北方供暖房 vs 南方潮湿房：同一盆植物浇水频次对照》

### Step 4: 价值自检（严格标准）

**对每个候选选题，必须回答**：

1. **能省时间吗？**
   - 能让用户不用花时间查资料？
   - 能让用户快速判断/操作？

2. **能省钱吗？**
   - 能避免买错东西？
   - 能避免植物养死重新买？

3. **能省力吗？**
   - 能减少试错次数？
   - 能简化操作步骤？

4. **能直接解决问题吗？**
   - 用户看完能立即行动？
   - 有明确的可执行方案？

**判断标准**：
- ✅ 至少满足2项 → 有价值
- ❌ 只满足1项或0项 → 重新选题

### Step 5: 去重检查

**检查已发布内容**：
- 这个植物最近写过几次？
- 这个角度最近用过吗？
- 这个问题最近解决过吗？

**去重规则**：
- 同一植物，7天内不重复
- 同一角度，14天内不重复
- 同一问题，30天内不重复（除非换角度）

### Step 6: 最终选题确定

**输出格式**：

```markdown
## 推荐选题

**选题**：[标题]

**用户焦虑点**：[从问题清单提取]

**时间节点**：[从营销日历提取]

**价值说明**：
- 省时间：[具体说明]
- 省钱：[具体说明]
- 省力：[具体说明]
- 直接解决：[具体说明]

**差异化**：
- 与已发布内容的差异：[具体说明]
- 独特角度：[具体说明]

**去重检查**：
- 最近30天相关话题：[列出]
- 本选题差异：[说明]
```

## 💡 灵活性的边界

### 什么是"灵活"？

**✅ 允许的灵活**：
1. 同一个用户问题，可以用不同角度包装
2. 可以结合时间节点，找到独特切入点
3. 可以用更有趣、更吸引人的方式表达
4. 可以从工具、方法、场景、对比等不同维度切入

**❌ 不允许的"灵活"**：
1. 完全脱离用户真实问题，自己想一个"有趣"的角度
2. 为了创新而创新，忽视实用价值
3. 选择用户不关心的小众话题
4. 用拟人化、情感化等手法包装，但没有实际价值

### 判断标准

**问自己**：
- 这个选题，用户会主动搜索吗？
- 这个选题，能解决用户正在焦虑的问题吗？
- 这个选题，看完后用户能立即行动吗？

如果3个问题都是"否"，说明太"灵活"了，需要调整。

## 🚫 常见错误示例

### 错误1：过度创意，脱离用户需求

**❌ 错误选题**：《植物也有"换季焦虑"？冬春交替期的5个信号》

**问题分析**：
- 拟人化角度有趣，但不是用户正在搜索的
- 用户不会搜"植物换季焦虑"
- 没有直接解决用户的具体问题
- 更像是科普，而不是解决方案

**✅ 改进方向**：
- 《冬春交替期，植物黄叶/掉叶是正常的吗？5个判断方法》
- 《1月底2月初，浇水频率要调整吗？北方vs南方对照表》

### 错误2：照搬用户问题，没有包装

**❌ 错误选题**：《为什么买回来一两个月就开始黄叶、掉叶、烂根？》

**问题分析**：
- 直接用用户问题做标题
- 没有吸引力
- 没有给出解决方向的暗示

**✅ 改进方向**：
- 《绿植买回来就黄叶？80%的人都忽略了这个问题》
- 《为什么你家绿植总养不活？3个最容易犯的错误》

### 错误3：只盯着高频问题，忽视其他角度

**❌ 错误思维**：
- 用户问题清单里有8个问题
- 我只能从这8个里选
- 黄叶、浇水、光照是高频，所以只写这些

**✅ 正确思维**：
- 8个问题是焦虑点，不是唯一选题
- 每个焦虑点可以延伸出多个角度
- 可以结合时间节点、场景、工具等维度
- 可以从预防、对比、反常识等角度切入

## 📝 实施要求

### 对 AI 的要求

**每次生成内容前，必须**：

1. 读取最新用户问题监测文件
2. 查询营销日历
3. 检查已发布内容
4. 生成3个候选选题（不同角度）
5. 对每个选题进行价值自检
6. 进行去重检查
7. 输出最终选题及理由

**禁止的行为**：

- ❌ 不读用户问题监测，直接想选题
- ❌ 只读用户问题监测，照搬问题
- ❌ 为了创意而创意，忽视实用价值
- ❌ 不做价值自检，自说自话
- ❌ 不做去重检查，重复话题
