# 🚀 快速启动指南

## 立即开始

### 第一次使用（推荐：小规模测试）

```bash
# 在 Claude Code 中执行
/content-production-pipeline 家居绿植 --count 1
```

**这会做什么：**
1. 生成10个选题（但只处理第1个）
2. 为这个选题生成小红书内容（1个文案 + 6张图片）
3. 大约需要 3-5 分钟

**预期输出：**
```

├── 选题库/
│   └── 2026-01-14_家居绿植_选题报告.md
```

## 常见问题

### Q1: 执行时间太长怎么办？
**A:** 使用 `--count` 参数减少处理数量
```bash
/content-production-pipeline 关键词 --count 3
```

### Q2: 某个选题生成失败怎么办？
**A:** 流水线会自动跳过失败的选题，继续处理下一个。失败原因会记录在执行报告中。

### Q3: 可以中途停止吗？
**A:** 可以，但已生成的内容会保留。建议使用 `--count` 参数控制数量。

### Q4: 生成的内容在哪里？
**A:**
- 选题报告：`/Users/dj/Documents/slowseasons AI工厂/选题库/`
- 小红书内容：`/Users/dj/Documents/slowseasons AI工厂/内容发布/发布记录/2026/小红书/`
- 执行报告：`/Users/dj/Documents/slowseasons AI工厂/内容发布/发布记录/2026/`

### Q5: 如何查看执行报告？
**A:** 执行完成后会自动生成报告，路径会在最后显示。

## 技巧和建议

### 💡 选择合适的关键词
- ✅ 具体场景：办公室绿植、阳台种植
- ✅ 明确人群：懒人养花、新手种植
- ✅ 季节时令：春季种植、冬季养护
- ❌ 太宽泛：植物、绿植

### 💡 合理安排时间
- 测试阶段：使用 `--count 1`
- 日常生产：使用 `--count 3`
- 批量储备：使用默认（10个）

### 💡 检查输出质量
- 查看生成的文案是否符合预期
- 检查配图是否与文案匹配
- 验证商品植入是否自然
