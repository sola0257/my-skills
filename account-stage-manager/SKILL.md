# 账号阶段管理器 v2.0

## 📋 概述

统一管理多平台账号的成长阶段。采用**双重维度判断机制**，不再单纯依赖粉丝数，而是结合真实互动情况来决定运营策略。

**核心升级：**
1. **双维度判断**：粉丝数 OR 评论互动（满足任一即可进阶）。
2. **新标准**：优化了各阶段的门槛，更符合 2026 年运营现状。

---

## 🎯 阶段划分标准

### 1. 小红书 & 视频号 & 抖音 & 快手

| 阶段 | 维度1：粉丝数 | 维度2：评论互动 | 内容/商品策略 |
|---|---|---|---|
| **起号期** | 0 - 500 | 评论区无/极少购买询问 | **纯干货，不挂车**。<br>建立信任，积累标签。商品仅作为道具出现。 |
| **成长期** | 500 - 2000 | 评论区出现"在哪买"、"求链接" | **软植入**。<br>1-2个要点提及商品，引导私信或看主页。 |
| **成熟期** | 2000+ | 评论区大量购买询问 | **正常带货**。<br>挂链接/小黄车，强种草，强调性价比/卖点。 |

### 2. 微信公众号

| 阶段 | 维度1：粉丝数 | 维度2：阅读/互动 | 内容/商品策略 |
|---|---|---|---|
| **起号期** | 0 - 1000 | 阅读量 < 200 | **深度干货**。<br>打造专业人设，不推硬广。 |
| **成长期** | 1000 - 5000 | 阅读量 > 500 | **文末软广**。<br>文章底部贴片广告或小程序卡片。 |
| **成熟期** | 5000+ | 阅读量 > 2000 | **专场团购**。<br>服务号推送专属优惠。 |

## 🔄 执行流程

### 1. 更新数据

用户输入粉丝数或互动情况：
> "/account-stage-manager 小红书 300粉 评论区已有求链接"

### 2. AI 判断逻辑

```python
def determine_stage(platform, followers, interaction):
    # 优先判断互动（互动权重 > 粉丝数）
    if "大量购买询问" in interaction:
        return "成熟期"
    if "求链接" in interaction or "在哪买" in interaction:
        return "成长期"
        
    # 其次判断粉丝数
    if followers >= 2000:
        return "成熟期"
    elif followers >= 500:
        return "成长期"
    else:
        return "起号期"
```

### 3. 输出策略

AI 输出当前阶段及对应策略：
```markdown
📊 **账号状态更新：小红书**
- 粉丝数：300（起号期标准）
- 互动：评论区已有求链接（成长期信号）
- **综合判定：成长期** 🎉

💡 **本周策略建议**：
- 可以开始尝试**软植入**。
- 在笔记中自然带出商品卖点（1-2个）。
- 评论区积极回复购买咨询，引导私信。
```
