## 📏 SKILL.md 编写规范

### ✅ 应该包含的内容
- 概述（做什么）
- 执行流程（SOP步骤）
- 调用方式（命令示例）
- 输出结构（文件组织）
- 依赖关系（调用什么）
- 配置信息（路径、参数）

### ❌ 不应该包含的内容
- 完整的 Python 代码（应放在 scripts/）
- 详细的 Prompt（应放在 prompts/）
- 大段的知识内容（应放在 knowledge/）
- API 调用细节（应放在 scripts/）
- 复杂的算法逻辑（应放在 scripts/）

### 📏 长度控制
- 目标：200-400 行
- 最多不超过 500 行
- 超过 500 行说明需要拆分
