# 小红书文案生成规范 v3.0

> **重要更新**：避免具体人设、标签结构优化、正文格式规范

---

## 🚫 核心禁令（必须遵守）

### 1. 禁止设定具体人设

**❌ 错误示例：**
- "去年被吐槽'连个花都养不活'"
- "我搬家时发现..."
- "作为一个租房党..."
- "我在上海的时候..."（北京除外）

**✅ 正确示例：**
- "很多人会遇到..."
- "对于居住空间有限的朋友..."
- "新手常见的问题是..."
- "大家可能会发现..."

**原则：**
- 使用中性化、通用化的表述
- 避免臆造发布者的具体经历、身份
- 地域可以使用"北京"，其他地域要排除
- 用"很多人"、"大家"、"新手"等泛指词汇
- 用客观描述代替主观经历

## 📝 标题与标签结构

### 视觉标题（封面大字）
- **字数限制**：≤20字（含标点和emoji）
- **作用**：情绪化、痛点向、吸引点击
- **示例**：3天改造阳台花园！春节来客人不丢脸了😭

### 标签（总数≤10个）

**标签组成：**
1. **长尾关键词标签**（3-5个）：SEO优化，帮助搜索
   - 格式：#核心词+场景词
   - 示例：#阳台花园改造 #春节布置 #新手养花 #室内绿植 #花园设计

2. **其他标签**（5-7个）：情绪词、平台热词
   - 示例：#春日居家 #慢生活 #治愈系 #氛围感 #生活仪式感

**标签公式：**
```
长尾关键词标签（3-5个）+ 情绪/场景标签（5-7个）= 总数≤10个
```

**示例：**
```
#阳台花园改造 #春节布置 #新手养花 #室内绿植 #花园设计 #春日居家 #慢生活 #治愈系 #氛围感 #生活仪式感
```

## 📄 正文格式规范

### ⚠️ 硬性要求

**禁止使用Markdown格式符号：**
- ❌ 不要使用 `**加粗**`
- ❌ 不要使用 `*斜体*`
- ❌ 不要使用 `## 标题`
- ❌ 不要使用 `- 列表`

**正确格式：**
- ✅ 直接使用纯文本
- ✅ 用emoji和符号标注（✅ ❌ ① ② ③）
- ✅ 用换行和空行分段
- ✅ 可以直接复制粘贴到小红书发布页面

**示例对比：**

❌ **错误（有Markdown格式）：**
```
## 第一步：清理

**清理清单：**
- 枯死的植物
- 破损的花盆
```

✅ **正确（纯文本）：**
```
第一步：清理 🧹

清理清单：
✅ 枯死的植物
✅ 破损的花盆
```

## 🎯 开头规范（避免具体人设）

### ❌ 错误开头
- "去年春节，我家花园被亲戚吐槽..."
- "作为一个养花3年的老手..."
- "我在北京租房的时候..."

### ✅ 正确开头
- "春节快到了，花园还是老样子？"
- "很多人会遇到这个问题..."
- "新手常见的困扰是..."

**开头公式：**
1. **共鸣式**：你是不是也...
2. **问题式**：为什么XX总是...
3. **场景式**：春节快到了...
4. **痛点式**：很多人在XX上卡很久...
