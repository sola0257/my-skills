# Step 0 初始化检查清单

> 基于 follower-engagement-guide.md 整合
> 每次生成内容前必须执行

---

## 执行步骤

### 1. 询问粉丝数（强制使用 AskUserQuestion 工具）

- [ ] 使用 AskUserQuestion 工具询问当前粉丝数
- [ ] 提供选项：0粉、1-100粉、101-500粉、501-2000粉、2000+粉、其他
- [ ] 记录具体数字（区间选中位数）

### 2. 读取历史记录

- [ ] 读取 `/Users/dj/Desktop/全域自媒体运营/内容发布/粉丝数记录_小红书.json`
- [ ] 如果文件不存在，创建新文件

### 3. 判断粉丝数变化

- [ ] 对比上次记录，计算涨粉数量
- [ ] 如果 ≥5天未涨粉，触发5天提醒机制

### 4. 生成个性化鼓励语

- [ ] 根据涨粉数量、坚持天数、最近内容动态生成
- [ ] 风格：真诚、具体、有温度
- [ ] 避免空洞的"加油"、"继续努力"
- [ ] 提供下一步建议

### 5. 5天未涨粉提醒（如果适用）

- [ ] 提示可能原因：选题、时间、标题/封面、内容质量
- [ ] 询问是否需要分析最近内容数据

### 6. 更新粉丝数记录

- [ ] 追加新记录到 JSON 文件
- [ ] 更新 `last_update` 和 `current_stage`

## 账号阶段判断

| 阶段 | 粉丝数 | 内容策略 | 商品策略 |
|------|--------|---------|---------|
| 起号期 | 0-500 | 纯干货建立信任 | 仅作案例提及，禁止价格 |
| 成长期 | 500-2000 | 价值+适度引导 | 软植入，评论区引导，可提价格 |
| 成熟期 | 2000+ | 信任+转化 | 可挂链接，可说价格，可直接推荐 |

## 常见问题

**Q: 如果用户选择"没有变化"，还需要生成鼓励语吗？**
A: 需要。但鼓励语的重点是"坚持"，而不是"涨粉"。

**Q: 如果是第一次使用，没有历史记录怎么办？**
A: 创建新的记录文件，初始化第一条记录。
