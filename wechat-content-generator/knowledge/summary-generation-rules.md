## 📋 概述 vs 摘要

### 重要区分

| 名称 | 位置 | 长度 | 用途 |
|------|------|------|------|
| **概述** | 标题后，引用块格式 `> 概述` | ≤25字 | 转发卡片、会话列表显示 |
| **摘要** | 正文章节 `## 摘要` | 不限 | 文章正文的一部分，概括全文 |

### 文章结构示例

```markdown
# 君子兰冬季养护：温差管理与烂心预防完全指南

![封面](cover.png)

> 温差管理+浇水技巧+烂心预防，科学养护君子兰

## 摘要

君子兰作为传统年宵花，冬季养护有其独特的生理需求。本文从温差管理、浇水技巧、烂心预防三个维度，系统讲解君子兰冬季养护的科学方法，帮助你养出健康开花的君子兰。

---

## 一、正文内容...
```

**说明**：
- **概述**（`> 温差管理+浇水技巧...`）：用于转发卡片，必须 ≤25字
- **摘要**（`## 摘要` 章节）：正文的一部分，可以较长，详细概括文章内容

## 📏 概述的作用

## 📏 概述的作用

### 展示位置

1. **转发卡片**：用户转发文章时，概述显示在卡片上
2. **公众号会话**：文章推送时，概述显示在会话列表
3. **搜索结果**：文章被搜索时，概述作为预览文本

### 重要性

- ✅ 影响点击率：好的概述能吸引用户点击
- ✅ 传达核心价值：快速告诉用户文章讲什么
- ✅ 提升转发率：清晰的概述让转发更有效

---

## 📏 长度要求

### 强制规范

| 平台 | 长度限制 | 说明 |
|------|---------|------|
| **微信公众号** | ≤25字 | 含标点符号，不含emoji |
| **转发卡片** | ≤20字 | 显示效果最佳 |
| **推荐长度** | 18-22字 | 平衡信息量和可读性 |

### 为什么是25字？

1. **转发卡片限制**：超过25字会被截断，显示"..."
2. **阅读体验**：20字左右一眼能看完，不需要滚动
3. **信息密度**：25字足够传达核心价值

## ✍️ 撰写原则

### 原则1：概括核心价值

**✅ 正确示例**：
```
温差管理+浇水技巧+烂心预防，科学养护君子兰
```
- 字数：22字
- 包含：3个核心要点 + 目标植物

**❌ 错误示例**：
```
君子兰作为传统年宵花，冬季养护有其独特的生理需求。本文从温差管理、浇水技巧、烂心预防三个维度...
```
- 字数：80+字
- 问题：过长，这应该是正文摘要，不是概述

### 原则2：使用"+"连接要点

**格式**：`要点1+要点2+要点3，目标结果`

**示例**：
- `选土+浇水+施肥，新手养好绿萝` （15字）
- `光照+温度+湿度，多肉安全过冬` （15字）
- `修剪+换盆+病虫害，月季春季管理` （16字）

### 原则3：包含时效性或场景

**时效性**：
- `春季换盆3步骤，植物长势旺` （14字）
- `冬季浇水5原则，避免烂根` （13字）

**场景性**：
- `阳台种菜入门，小白也能收获` （14字）
- `室内养花指南，净化空气又美观` （15字）

### 原则4：避免空洞表述

**❌ 禁止**：
- "本文介绍了..."
- "详细讲解..."
- "全面分析..."
- "深度解读..."

**✅ 推荐**：
- 直接说要点
- 突出实用性
- 强调结果

## 📝 生成流程

### Step 1：提取核心要点

从文章中提取3-5个核心要点：
```
文章：君子兰冬季养护
核心要点：
1. 温差管理
2. 浇水技巧
3. 烂心预防
4. 光照需求
5. 花期管理
```

### Step 2：选择最重要的3个

根据文章重点，选择最重要的3个：
```
选择：
1. 温差管理（最重要）
2. 浇水技巧（常见问题）
3. 烂心预防（痛点）
```

### Step 3：组合成概述

使用"+"连接，添加目标结果：
```
温差管理+浇水技巧+烂心预防，科学养护君子兰
```

### Step 4：检查字数

```
字数统计：22字 ✅
是否 ≤25字：是 ✅
是否清晰：是 ✅
```

### Step 5：编写正文摘要

概述完成后，再编写正文的摘要章节（可以较长）：
```markdown
## 摘要

君子兰作为传统年宵花，冬季养护有其独特的生理需求。本文从温差管理、浇水技巧、烂心预防三个维度，系统讲解君子兰冬季养护的科学方法，帮助你养出健康开花的君子兰。
```

## 🎯 不同类型文章的概述模板

### 类型1：养护指南类

**模板**：`[要点1]+[要点2]+[要点3]，[目标结果]`

**示例**：
- `温差管理+浇水技巧+烂心预防，科学养护君子兰` （22字）
- `选土+浇水+施肥，新手养好绿萝` （15字）

### 类型2：问题解决类

**模板**：`[问题]+[解决方案]，[效果]`

**示例**：
- `叶子发黄？3招让绿萝恢复生机` （15字）
- `多肉徒长怎么办？控水+补光见效` （16字）

### 类型3：季节主题类

**模板**：`[季节]+[植物]+[核心操作]，[目标]`

**示例**：
- `春季换盆3步骤，植物长势旺` （14字）
- `冬季浇水5原则，避免烂根` （13字）

### 类型4：新手入门类

**模板**：`[植物]+[难度]+[核心要点]，[承诺]`

**示例**：
- `阳台种菜入门，小白也能收获` （14字）
- `多肉养护基础，新手0失败` （12字）

## ⚠️ 常见错误

### 错误1：过长

**❌ 错误**：
```
概述：君子兰作为传统年宵花，冬季养护有其独特的生理需求。本文从温差管理、浇水技巧、烂心预防三个维度，系统讲解君子兰冬季养护的科学方法，帮助你养出健康开花的君子兰。
```
- 字数：80+字
- 问题：远超25字限制，这应该是正文摘要，不是概述

**✅ 修正**：
```
概述：温差管理+浇水技巧+烂心预防，科学养护君子兰
正文摘要：君子兰作为传统年宵花，冬季养护有其独特的生理需求。本文从温差管理、浇水技巧、烂心预防三个维度，系统讲解君子兰冬季养护的科学方法，帮助你养出健康开花的君子兰。
```
- 概述：22字 ✅
- 正文摘要：保留完整内容 ✅

### 错误2：空洞

**❌ 错误**：
```
本文详细介绍了君子兰的养护方法
```
- 字数：16字
- 问题：没有具体信息

**✅ 修正**：
```
温差管理+浇水技巧+烂心预防，科学养护君子兰
```
- 字数：22字
- 优点：具体要点清晰

### 错误3：重复标题

**❌ 错误**：
```
标题：君子兰冬季养护：温差管理与烂心预防完全指南
概述：君子兰冬季养护完全指南
```
- 问题：概述重复标题，没有增量信息

**✅ 修正**：
```
标题：君子兰冬季养护：温差管理与烂心预防完全指南
概述：温差管理+浇水技巧+烂心预防，科学养护君子兰
```
- 优点：概述补充具体要点

### 错误4：混淆概述和摘要

**❌ 错误**：
```markdown
# 标题

> 君子兰作为传统年宵花，冬季养护有其独特的生理需求...（80字）

---

正文...
```
- 问题：把正文摘要放在了概述位置

**✅ 修正**：
```markdown
# 标题

> 温差管理+浇水技巧+烂心预防，科学养护君子兰

## 摘要

君子兰作为传统年宵花，冬季养护有其独特的生理需求...

---

正文...
```
- 优点：概述和摘要各司其职

## 🔧 技术实现

### Markdown 格式

```markdown
# 文章标题

![封面](cover.png)

> 概述内容（≤25字）

## 摘要

[正文摘要内容，可以较长，详细概括文章]

---

正文内容...
```

**说明**：
- **概述**（`> 概述内容`）：≤25字，用于转发卡片
- **摘要**（`## 摘要` 章节）：正文的一部分，可以较长

### 代码提取逻辑

```python
def extract_summary(markdown_content):
    """提取概述（≤25字）用于转发卡片"""
    # 查找引用块（> 开头的行）
    quote_pattern = r'^>\s*(.+)$'
    match = re.search(quote_pattern, markdown_content, re.MULTILINE)

    if match:
        summary = match.group(1).strip()
        # 限制25字
        if len(summary) > 25:
            summary = summary[:22] + "..."
        return summary

    return ""
```

**说明**：
- 提取的是 `> 概述内容`，不是 `## 摘要` 章节
- 用于转发卡片和会话列表显示
- 正文的 `## 摘要` 章节保持不变

### 验证检查

```python
def validate_summary(summary):
    """验证摘要是否符合规范"""
    if not summary:
        return False, "摘要不能为空"

    if len(summary) > 25:
        return False, f"摘要过长（{len(summary)}字），应≤25字"

    if len(summary) < 10:
        return False, f"摘要过短（{len(summary)}字），建议≥10字"

    return True, "摘要符合规范"
```

## 📌 总结

**核心要求**：
1. **长度**：≤25字（推荐18-22字）
2. **格式**：`要点1+要点2+要点3，目标结果`
3. **内容**：具体要点，不空洞
4. **位置**：标题之后，引用块格式 `> 概述`
5. **区分**：概述（≤25字）≠ 摘要（正文章节，不限长度）

**生成流程**：
1. 提取核心要点
2. 选择最重要的3个
3. 使用"+"连接
4. 添加目标结果
5. 检查字数
6. 编写正文摘要（保留）

**质量标准**：
- 字数符合要求
- 信息密度高
- 清晰易懂
- 有吸引力
- 不与正文摘要混淆
