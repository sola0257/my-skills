# 内容生成工作流管理器 v1.0

**版本**：v1.0
**创建日期**：2026-01-30
**核心功能**：统一管理所有平台的内容生成流程，确保执行一致性

---

## 🎯 核心定位

**问题**：
- 用户每次生成内容时，需要手动调用不同的 skill
- 不同 skill 的执行流程可能不一致
- 容易遗漏必需步骤（如去重检查）

**解决方案**：
- 提供统一入口
- 自动执行标准流程
- 根据平台类型调度对应的 content-generator

---

## 🚨 执行协议

**执行规则：**
- ✅ **必须** 遵循统一的8步工作流
- ✅ **必须** 先进行苏格拉底式提问
- ✅ **必须** 检查已发布内容（去重）
- ✅ **必须** 根据账号阶段决定商品植入策略
- ✅ **必须** 调用对应平台的 content-generator

---

## 📋 核心功能

1. **统一入口**：用户只需调用一个 skill
2. **标准流程**：自动执行8步工作流
3. **智能调度**：根据平台类型调用对应的 generator
4. **流程监控**：确保每个步骤都正确执行
5. **错误处理**：如果某个步骤失败，提供明确的错误信息

---

## 🔄 执行流程

### Step 0: 苏格拉底式提问 [强制]

**使用工具**：`AskUserQuestion`

**必须询问的信息**：

```
问题1：请选择内容平台
选项：
- 小红书
- 微信订阅号
- 微信服务号
- 视频号
- 抖音
- 快手
- 全平台（一次生成多个平台）

问题2：内容目的是什么？
选项：
- 涨粉（纯干货，建立信任）
- 引流（引导关注/添加）
- 推销商品（直接转化）
- 纯干货分享（无商业目的）

问题3：是否有特定选题？
选项：
- 是（请描述选题）
- 否（需要AI根据营销日历推荐）

问题4：是否需要植入商品？
选项：
- 是（需要商品推荐）
- 否（纯内容）
- 极少（仅1-2处自然提及）
```

**特殊询问**（根据平台类型）：
- 如果是"微信订阅号" → 额外询问：长文 or 图文？
- 如果是"视频" → 额外询问：知识分享类 or 生活感悟类？时长？

### Step 1: 检查已发布内容 [强制]

**操作步骤**：

1. **确定检查路径**
   ```python
   platform_paths = {
       "小红书": "/Users/dj/Desktop/全域自媒体运营/内容发布/发布记录/2026/小红书/",
       "订阅号": "/Users/dj/Desktop/全域自媒体运营/内容发布/发布记录/2026/订阅号/",
       "服务号": "/Users/dj/Desktop/全域自媒体运营/内容发布/发布记录/2026/服务号/",
       "视频号": "/Users/dj/Desktop/全域自媒体运营/内容发布/发布记录/2026/视频号/",
       "抖音": "/Users/dj/Desktop/全域自媒体运营/内容发布/发布记录/2026/抖音/",
       "快手": "/Users/dj/Desktop/全域自媒体运营/内容发布/发布记录/2026/快手/"
   }
   ```

2. **读取最近30天的内容**
   - 使用 Bash 工具：`ls -lt | head -30`
   - 提取标题和主题

3. **分析重复性**
   - 统计话题频率
   - 识别重复角度

4. **输出去重报告**
   ```
   📊 已发布内容分析（最近30天）：

   高频话题：
   - 蝴蝶兰养护：3篇（浇水、黄叶、烂根）
   - 仙客来养护：2篇（温度、浇水）
   - 年宵花选购：1篇

   ⚠️ 建议避开：
   - 蝴蝶兰、仙客来、年宵花的常规养护话题

   ✅ 建议方向：
   - 其他植物品种（多肉、观叶植物）
   - 春季换盆、病虫害防治
   - 场景化内容（办公室、卧室）
   ```

---

### Step 2: 确定选题 [强制]

**情况A：用户指定选题**
1. 检查是否与已发布内容重复
2. 如果重复，提醒用户并建议调整角度
3. 用户确认后继续

**情况B：AI推荐选题**
1. 调用 `/marketing-calendar` 查询当前时间节点
2. 结合去重分析，排除已发布的话题
3. 推荐3个选题供用户选择
4. 每个选题说明：
   - 时间节点相关性
   - 与已发布内容的差异化
   - 预期效果（涨粉/引流/转化）

**输出示例**：
```
📝 推荐选题（基于1月31日营销日历）：

选题1：立春前的植物准备工作
- 时间节点：距离立春还有4天
- 差异化：未发布过"立春"相关内容
- 预期效果：涨粉（实用干货）

选题2：办公室绿植选择指南
- 时间节点：节后复工季
- 差异化：场景化内容，未发布过
- 预期效果：涨粉+引流

选题3：多肉植物春季养护
- 时间节点：春季来临
- 差异化：品类差异，未发布过多肉内容
- 预期效果：涨粉

请选择一个选题，或提供你自己的选题。
```

---

### Step 3: 调用对应平台的 Content Generator [强制]

**根据用户选择的平台，调用对应的 Skill**：

| 平台 | 调用的 Skill | 传递的参数 |
|------|-------------|-----------|
| 小红书 | `xiaohongshu-content-generator` | 选题、目的、商品植入 |
| 微信订阅号 | `wechat-content-generator` | 选题、格式、目的、商品植入 |
| 微信服务号 | `wechat-content-generator` | 选题、格式、目的、商品植入 |
| 视频号/抖音/快手 | `video-script-generator` | 选题、类型、时长、目的、商品植入 |

**调用方式**：
```python
# 使用 Skill tool
Skill(
    skill="xiaohongshu-content-generator",
    args=f"选题：{topic}，目的：{purpose}，商品植入：{product_placement}"
)
```

**重要**：
- 传递的参数包含 Step 0-2 收集的所有信息
- 对应的 generator 会跳过 Step 0-2，直接从 Step 3 开始执行
- 确保信息传递完整

---

### Step 4-8: 由对应的 Generator 执行

**这些步骤由对应平台的 content-generator 自动执行**：
- Step 4: 知识搜索
- Step 5: 商品匹配（条件执行）
- Step 6: 生成内容
- Step 7: 生成配图
- Step 8: 保存文件/推送

## 🚫 错误处理

**如果某个步骤失败**：

1. **明确错误信息**
   ```
   ❌ Step 1: 检查已发布内容 - 失败
   错误原因：无法读取目录 /Users/dj/Desktop/全域自媒体运营/内容发布/发布记录/2026/小红书/
   建议：请检查目录是否存在
   ```

2. **提供解决方案**
   - 如果是目录不存在 → 创建目录
   - 如果是权限问题 → 提示用户修改权限
   - 如果是 Skill 调用失败 → 检查 Skill 是否存在

3. **询问是否继续**
   - 如果是非关键步骤（如去重检查失败） → 询问是否跳过
   - 如果是关键步骤（如 Skill 调用失败） → 必须解决后才能继续
