# 自动学习与改进机制 - 用户指南

## 🎯 核心理念

**你不需要做任何事情，系统会自动学习和改进。**

**适用范围**：✅ **所有 Skills**（不仅限于内容生成类）

---

## 🔄 自动化流程

### 第1次出现问题

```
你执行 Skill
    ↓
我可能跳过某个步骤（如网络搜索）
    ↓
Hook 自动检测到问题
    ↓
自动记录到 instincts.json
    ↓
你看到警告："⚠️ 缺少必需步骤: 网络搜索"
```

**你不需要做任何事情**，问题已被记录。

---

### 第2次执行相同 Skill

```
你再次执行 Skill
    ↓
我自动读取 instincts.json
    ↓
看到提醒："上次跳过了网络搜索"
    ↓
我特别注意，确保执行网络搜索
    ↓
Hook 再次检测
    ↓
如果改进 → 标记为已解决
如果未改进 → 升级警告级别
```

**你不需要做任何事情**，系统自动改进。

---

### 第3次相同问题

```
如果第3次还是相同问题
    ↓
我自动阅读 execution-quality-assurance.md
    ↓
按照四层机制自动修正
    ↓
或者提示你需要手动修正 SKILL.md
```

**只有在第3次还是有问题时，才需要你介入。**

---

## 💬 简化的用户指令

### 如果你想主动触发修正

**简短版本**：
```
修正 [skill-name] 的执行问题
```

**完整版本**（可选）：
```
我发现 [skill-name] 跳过了 [某个步骤]，请修正
```

**我会自动**：
1. 读取 instincts.json 查看历史问题
2. 阅读 execution-quality-assurance.md
3. 按照四层机制修正问题

---

## 📊 查看学习记录

如果你想查看系统学到了什么：

```
查看 instincts.json
```

或者：

```
cat ~/.claude/skills/_global_config/instincts.json
```
